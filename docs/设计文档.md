# core

下方为 core 的配置结构

```ts
interface Config {
  /**
   * 页面模版
   */
  templates: Template[];
  /**
   * 内部接口的实现
   */
  service: {
    newEntity: NewEntity;
    deleteEntity: DeleteEntity;
    getEntity: GetEntity;
    getEntitys: GetEntitys;
    patchEntity: PatchEntity;
    addField: AddField;
    deleteField: DeleteField;
    updateField: UpdateField;
    // MoveField: MoveField;
    setPrimaryKey: SetPrimaryKey;
    addPage: AddPage;
    getPage: GetPage;
    updatePage: UpdatePage;
    deletePage: DeletePage;
    movePage: MovePage;
  };
  request: AxiosInstance;
}
```

使用一个`core`需要配置好模版、接口、`Axio`实例。

## 模版

模版是由一个`plugin`实例化实现的,他提供了配置并渲染一个页面的功能

## 接口

所有需要持久化配置的接口,需要根据接口类型实现所有的接口

## Axio 实例

默认所有`plugin`使用它去请求调用,你可以在实例中加中间件实现数据转换等功能。

# render

根据 core 渲染出

- 实体工作台(即从创建实体->新建字段/页面)
- 模版工作台(即配置页面的工作台)
- 模版页面(即最终需要展示的页面)

# plugin

所有页面的配置需要配哪些,如何根据配置展示都是由插件来描述的
插件需要同时实现下面的功能

- 如何配置
- 如何展示
